---
title: Wallets & signing
description: How transactions built by Printr get signed.
---

Printr builds transactions; you choose how they get signed. Both supported paths are
human-gated — you stay in control of every signature.

## Browser signer

You approve each transaction in a browser wallet, one transaction at a time. Printr runs a
local signing server and opens the wallet for confirmation.

## Self-custody keystore

An encrypted key you provision and unlock locally. Once unlocked, signing tools use it for the
session.
