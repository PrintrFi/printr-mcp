---
title: Creating tokens
description: Launch a token on an EVM chain or Solana.
---

A launch has two halves: **build** an unsigned payload from your metadata, then **sign and
submit** it. This guide covers the build; see [Wallets & signing](/docs/wallets-signing) for the
second half.

## Supported chains

Chains are identified by [CAIP-2](https://chainagnostic.org/CAIPs/caip-2). Pass these ids in the
`chains` array. This table is generated from the SDK's chain metadata, so it always matches what
`@printr/sdk` actually supports.

<SupportedChains />

A token may target multiple chains in one launch — pass several CAIP-2 ids in `chains`.

## Walkthrough

<Steps>

<Step>

### Estimate the cost (optional)

```ts twoslash
import { createPrintrClient, env } from "@printr/sdk";

const client = createPrintrClient({
  apiKey: env.PRINTR_API_KEY,
  baseUrl: env.PRINTR_API_BASE_URL,
});
```

Use the `printr_quote` MCP tool (or the API) to itemize per-chain cost before committing.

</Step>

<Step>

### Prepare the image

- Max size **500KB** (base64).
- **JPEG recommended** — PNGs are usually too large; convert first.
- Pass either `image` (base64/URL) or `image_path` (local file, resolved and compressed for you).

</Step>

<Step>

### Build the unsigned payload

```ts twoslash
import { createPrintrClient, env } from "@printr/sdk";
const client = createPrintrClient({ apiKey: env.PRINTR_API_KEY, baseUrl: env.PRINTR_API_BASE_URL });
// ---cut---
import { buildToken } from "@printr/sdk";

const result = await buildToken(
  {
    creator_accounts: ["eip155:8453:0xYourAddress"],
    name: "My Token",
    symbol: "MINE",
    description: "A cross-chain token.",
    chains: ["eip155:8453"],
    initial_buy: { spend_usd: 10 },
    image_path: "./logo.jpg",
  },
  client,
);
```

`buildToken` returns a `ResultAsync` with the unsigned transaction payload.

</Step>

<Step>

### Sign and submit

Hand the payload to a signer — browser or keystore. See
[Wallets & signing](/docs/wallets-signing).

</Step>

</Steps>

<Callout type="info">
  Where creator fees go is controlled by `fee_sink`
  (`dev` | `stake_pool` | `buyback` | `liquidity_pool`), defaulting to `dev` (the creator wallet).
</Callout>
